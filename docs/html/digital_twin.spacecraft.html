update_subsystems(old_mode: int, new_mode: int, rv: ndarray, com_window: bool, eclipse_status: bool, delta_t: TimeDelta, r_earth_sun: Annotated[Quantity, PhysicalType('length')], gs_coords: ndarray | None) → None[source]

Update all subsystems and spacecraft properties based on current conditions.

Parameters:

  • old_mode (int) – Previous operational mode of the spacecraft.

  • new_mode (int) – New operational mode of the spacecraft.

  • rv (np.ndarray) – Position and velocity vector of the spacecraft (km).

  • com_window (bool) – Indicates whether the spacecraft is in a communication window.

  • eclipse_status (bool) – Indicates whether the spacecraft is in Earth’s shadow.

  • delta_t (TimeDelta) – Time step for the update.

  • r_earth_sun (Quantity["length"]) – Vector from Earth to the Sun.

  • gs_coords (Optional[np.ndarray]) – Coordinates of the ground station if satellite is in visibility window.

Updates:

  • Individual updates for each subsystem.

  • EPS battery level.

  • Data storage.

  • Safe flags.

digital_twin.spacecraft.subsystem module

Subsystem Interface Module

This module defines the SubSystem class, which serves as an abstract interface for all spacecraft subsystems.

class digital_twin.spacecraft.subsystem.SubSystem[source]

Bases: object

Abstract interface for spacecraft subsystems.

This class ensures that all subsystems implement the necessary methods to interact with the spacecraft. Each subsystem is expected to define its behavior for updates, power consumption, safety flags, and identification.

Methods defined here must be implemented by all derived classes.
